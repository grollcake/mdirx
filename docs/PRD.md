## 1. 개요

### 1.1 비전
**"Mdir, reborn on macOS."** — DOS 시대의 Mdir/NCD와 Windows의 NexusFile이 보여준 키보드 중심·듀얼 패널 파워유저 경험을 macOS 네이티브로 부활시킨다.

### 1.2 한 줄 소개
MdirX는 macOS용 듀얼 패널 파일 매니저로, NexusFile과 호환되는 단축키·기능 체계를 SwiftUI 네이티브로 구현한다.

### 1.3 포지셔닝
- Finder의 한계(단일 패널, 키보드 약함, 일괄 작업 빈약)에 답답함을 느끼는 사용자를 위한 대안.
- Path Finder·ForkLift 같은 상용 대안 대비 **"NexusFile 출신 사용자의 근육 기억을 그대로"** 살리는 데 집중.

---

## 2. 문제 정의

### 2.1 사용자 통점
1. macOS Finder는 듀얼 패널이 없어 파일 이동·비교 작업이 비효율적이다.
2. 한국 파워유저(40~50대 중심)는 DOS Mdir → Windows NexusFile 계보의 키바인딩에 익숙한데, macOS에서 동일 경험을 주는 도구가 없다.
3. 기존 macOS 듀얼 패널 앱은 키보드 중심 워크플로우(F10 트리 점프, 함수키 커스텀, 패턴 선택)가 약하다.
4. Spotlight 통합·QuickLook·태그 같은 macOS 자산을 살리면서도 NCD 스타일 트리 점프를 동시에 제공하는 앱이 없다.

### 2.2 기회
- 한국 시장: NexusFile 사용자가 Mac으로 전환할 때 직접 대체재가 없음.
- 글로벌 시장: Norton/Total Commander 후예에 익숙한 파워유저층.

## 3. 타겟 사용자

### 3.1 1차 페르소나 — "전직 NexusFile 사용자 P씨"
- 40대 개발자/디자이너, Windows에서 Mac으로 전환.
- F10·F11·Tab 패널 스위치·⇧⌥R이 손가락에 새겨져 있음.
- Finder가 "느리고 답답하다"고 느끼며 매일 수십~수백 파일을 이동·정리.

### 3.2 2차 페르소나 — "커맨더 패밀리 사용자 G씨"
- Total Commander / Midnight Commander 사용자, macOS 신규 유입.
- 듀얼 패널·키보드 워크플로우만 잘 작동하면 브랜드 무관.

### 3.3 비대상
- Finder로 충분히 만족하는 일반 사용자.
- 클라우드 드라이브 GUI·미디어 관리 중심 사용자.

---

## 4. 목표 / 비목표

### 4.1 목표 (M5 = 1.0 시점)
- G1. NexusFile 사용자가 **재학습 없이 즉시 사용 가능한 키바인딩 호환성** 제공.
- G2. macOS 15 네이티브 성능 — 10,000개 항목 폴더에서 60fps 스크롤.
- G3. **듀얼 패널·탭·F10 점프·고급 이름변경·ZIP·QuickLook** 5대 핵심 기능을 정확히 동작시킨다.
- G4. Developer ID 서명 + 공증 통과한 DMG로 직접 배포.
- G5. 한국어/영어 동시 로컬라이즈.

### 4.2 비목표 (1.0 범위 밖)
- FTP / SFTP / WebDAV 클라이언트
- 7z·rar·tar.gz 등 ZIP 외 압축 포맷
- iOS / iPadOS / Windows / Linux 지원
- 클라우드 동기화 자체 구현 (iCloud/Dropbox 감지·표시는 OK)
- 미디어 뷰어·편집기 (QuickLook 위임)
- App Store 배포

## 6. 기능 요구사항

### 6.1 레이아웃
- FR-1.1 단일/듀얼 패널 토글 (`⌘\`)
- FR-1.2 패널별 독립 탭, 최소 1개·최대 무제한 (실용 한계 ~32)
- FR-1.3 좌측 사이드바: 즐겨찾기·iCloud·외장 볼륨·태그
- FR-1.4 활성 패널 시각 인디케이터 + Tab으로 전환
- FR-1.5 상태바: 항목 수·선택 수·총 크기·진행 작업

### 6.2 파일 작업
- FR-2.1 복사/이동/붙여넣기 (⌘C·⌘X·⌘V), 즉시 복사/이동(⌥C·⌥M)
- FR-2.2 3단계 삭제: 휴지통(⌫) / 완전삭제(⇧⌘⌫) / 0-덮어쓰기(⌃⇧⌘⌫)
- FR-2.3 새 폴더(⌘⇧N), 이름변경(Return)
- FR-2.4 고급 이름변경(⇧⌥R) — 패턴/정규식/번호매김/대소문자, 라이브 프리뷰
- FR-2.5 패턴 선택 (글롭/정규식), 전체 선택, 역선택(⌥U)
- FR-2.6 모든 장기 작업은 백그라운드 큐, 취소·일시정지 가능
- FR-2.7 충돌 시 정책: 덮어쓰기/스킵/이름 변경(자동)/물어보기

### 6.3 네비게이션
- FR-3.1 주소표시줄에서 직접 경로 입력, 자동완성, 드롭다운 히스토리
- FR-3.2 뒤로/앞으로(←/→ 또는 ⌘[ ⌘])
- FR-3.3 부모 디렉터리(⌘↑)
- FR-3.4 F10 폴더 점프 다이얼로그 — 트리 + 퍼지 검색
- FR-3.5 F11 즐겨찾기 — 추가·제거·재정렬·이름 지정
- FR-3.6 작업폴더(워킹 디렉터리) 지정 기능

### 6.4 검색
- FR-4.1 폴더 내 검색(⌘F) — 이름·확장자·크기·날짜 필터
- FR-4.2 Spotlight 글로벌 검색(⌘⇧F) — `NSMetadataQuery`
- FR-4.3 검색 결과를 활성 패널에 가상 폴더로 표시

### 6.5 압축
- FR-5.1 선택 항목을 ZIP으로 압축 (메뉴 + 단축키)
- FR-5.2 ZIP 해제(⌘E)
- FR-5.3 지능형 해제(⌥Q) — 단일 폴더 내포 시 자동 평탄화
- FR-5.4 ZIP 내부를 가상 폴더처럼 탐색·미리보기(읽기 전용)

### 6.6 미리보기
- FR-6.1 Space — QuickLook 패널
- FR-6.2 썸네일(이미지/PDF/비디오) — `QLThumbnailGenerator` + LRU 캐시
- FR-6.3 폴더 정보 패널(선택 항목 메타데이터, EXIF)

### 6.7 변경 감지
- FR-7.1 활성 폴더에 FSEvents 모니터, 외부 변경 시 자동 반영
- FR-7.2 새 항목은 부드러운 페이드 인 애니메이션

### 6.8 사용자 정의
- FR-8.1 F1~F12에 명령 바인딩 (명령 카탈로그에서 선택)
- FR-8.2 단축키 전체 커스터마이저
- FR-8.3 경로 복사 옵션: 전체경로·파일명·폴더경로
- FR-8.4 심볼릭 링크 표시·해석 옵션
- FR-8.5 숨김 파일 토글(⌘⇧.)
- FR-8.6 스킨/테마: 라이트·다크·사용자 컬러 토큰
- FR-8.7 설정 import/export (JSON)

### 6.9 명령 팔레트
- FR-9.1 ⌘⇧P — 모든 명령을 퍼지 검색해 실행
- FR-9.2 명령 카탈로그는 함수키 바인딩과 단일 소스 공유

### 6.10 로컬라이즈
- FR-10.1 한국어/영어 동시 지원
- FR-10.2 메뉴에서 언어 전환 (재시작 없이)

## 7. 비기능 요구사항

### 7.1 성능
- NFR-1.1 10,000개 항목 폴더 진입 < 500ms (SSD)
- NFR-1.2 항목 스크롤 60fps 유지
- NFR-1.3 콜드 스타트 < 1.5s
- NFR-1.4 큰 파일 복사 시 UI 무블로킹

### 7.2 안정성
- NFR-2.1 파일 작업 중 크래시 시 작업 큐 상태 복구
- NFR-2.2 0-덮어쓰기 등 파괴적 작업은 반드시 확인 다이얼로그

### 7.3 호환성
- NFR-3.1 macOS 15 Sequoia 이상
- NFR-3.2 Intel·Apple Silicon Universal 빌드
- NFR-3.3 다크모드·라이트모드 동시 지원
- NFR-3.4 한국어 파일명·NFD/NFC 정규화 처리

### 7.4 보안·프라이버시
- NFR-4.1 비샌드박스이나 첫 실행 시 Full Disk Access 안내
- NFR-4.2 외부 네트워크 통신 없음 (FTP 제외 결정)
- NFR-4.3 텔레메트리·분석 도구 없음

### 7.5 접근성
- NFR-5.1 VoiceOver 기본 호환
- NFR-5.2 키보드만으로 모든 기능 도달 가능
- NFR-5.3 동적 텍스트 크기 일부 지원

### 7.6 배포
- NFR-6.1 Developer ID 서명 + Apple Notarization 통과
- NFR-6.2 DMG + Sparkle 자동 업데이트
- NFR-6.3 단일 사용자 라이선스 (오픈소스 결정 시 적용 라이선스 명시)

## 8. 성공 지표

### 8.1 정량
- M-1. 1.0 출시 후 3개월 내 DAU 500+ (배포 채널 분석)
- M-2. 평균 세션 시간 15분 이상
- M-3. 키바인딩 호환성 자가 평가: NexusFile 핵심 단축키 100% 매핑
- M-4. 크래시-프리 세션 비율 > 99.5%

### 8.2 정성
- Q-1. NexusFile 출신 사용자의 "근육 기억으로 바로 쓸 수 있다" 피드백
- Q-2. macOS 15 디자인 가이드라인 외관 일관성 평가 통과
- Q-3. App Store 외 채널(블로그, 커뮤니티) 자발 리뷰

## 9. 의존성 / 제약

- Xcode 16+ 필요
- Swift 6 strict concurrency 모드 적용
- AppleArchive, QuickLookUI, NSMetadataQuery, FSEvents 등 시스템 프레임워크에 의존
- 비샌드박스이므로 App Store 배포 불가 — 직접 배포 인프라(DMG 호스팅, Sparkle 서버) 필요

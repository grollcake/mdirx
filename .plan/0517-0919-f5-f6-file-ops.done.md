# F5/F6 다른 패널로 복사·이동

## 사용자 요구 (정제)
- 현재 듀얼 패널과 다중 선택이 동작하므로, NexusFile 식 `F5` 복사와 `F6` 이동을 붙인다.
- 활성 패널의 선택 항목(선택이 없으면 cursor 항목)을 반대 패널의 현재 경로로 복사·이동해야 한다.
- 드라이브/볼륨, 부모 링크 같은 선택 불가 행은 작업 대상에서 제외하고, 같은 이름 충돌은 자동 덮어쓰기하지 않는다.

## 개요
파일 목록에서 실제 파일을 다른 패널로 보내는 M1 핵심 파일 작업을 구현한다. 이번 범위는 단발 복사·이동과 충돌 방지까지이며, 삭제·진행률 큐·Undo·덮어쓰기 정책 UI는 후속 계획으로 둔다.

## 요구사항
- `F5`는 활성 패널의 대상 항목을 반대 패널 디렉터리로 복사한다.
- `F6`은 활성 패널의 대상 항목을 반대 패널 디렉터리로 이동한다.
- selection이 있으면 selection 전체를 사용하고, selection이 비어 있으면 cursor가 선택 가능한 항목일 때 그 1개를 사용한다.
- mounted volume, 부모 링크, 선택 불가 행은 대상에서 제외한다.
- 대상 경로에 같은 이름이 있으면 작업을 시작하지 않고 오류를 표시한다.
- 성공 후 양쪽 패널을 reload하고 selection/cursor가 깨지지 않아야 한다.

## 수도 코드
```text
on F5:
  source = activePane
  target = otherPane
  urls = source.operationItemURLs()
  if urls is empty: return
  fs.copyItems(urls, to: target.currentURL)
  reload source and target

on F6:
  source = activePane
  target = otherPane
  urls = source.operationItemURLs()
  if urls is empty: return
  fs.moveItems(urls, to: target.currentURL)
  reload source and target

copy/move in FileSystemActor:
  assert destination is directory
  preflight every destination URL
  if any destination exists: throw
  perform FileManager copyItem/moveItem sequentially
```

## 아키텍처
- `Core/FileSystem/FileSystemActor.swift`: `copyItems(_:to:)`, `moveItems(_:to:)` 추가.
- `Features/DualPane/PaneState.swift`: 현재 작업 대상 URL 계산 헬퍼 추가.
- `Features/DualPane/BrowserSession.swift`: 활성/반대 패널 판정과 copy/move orchestration 추가.
- `Features/DualPane/DualPaneView.swift`: `F5`/`F6` key handling 추가.
